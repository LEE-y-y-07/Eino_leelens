# 缺陷说明:工具未绑定到模型

## 变更记录表

| 日期 | 版本 | 作者 | 变更内容 |
|---|---|---|---|
| 2026-05-07 | v1.0 | AI | 首次提交 |

## 基本信息

- **编号**:001
- **标题**:工具未绑定到模型(Tools not bound to ChatModel)
- **严重级别**:严重(产品价值为零)
- **影响范围**:所有通过 `ProxyChatModel` 路径调用 LLM 的 Agent
- **发现日期**:2026-05-07

## 问题现象

1. 用户触发任何基于 Agent 的任务(目录分析、文档生成等),任务状态显示为 `succeeded`,生成的文档也保存进数据库
2. 但生成的文档内容与真实仓库代码**完全不符**:
   - 子模块名被编造(如真实有 `springai_alibaba` 却写成 `com.example.SpringAIApplication`)
   - 包名错误(真实 `com.atguigu.ai.*`,模型写 `com.example`)
   - 文档不引用任何真实代码文件、类名、方法名
3. 模型调用日志显示 `tool_count=0 tool_names=[]`
4. 针对一个 9 子模块的 Spring Boot 项目,API Key 调用统计只累计 **6~11 次**(正常情况应为 50~200+ 次)

## 复现步骤

已用三个不同模型验证,均复现相同现象:

1. 启动服务,配置任意 OpenAI 兼容模型(`provider: openai`)的 API Key
2. 通过 `POST /api/repositories` 创建仓库,Clone 完成后自动触发 `TocWriter`
3. 等待任务跑完,调用 `GET /api/documents/:id` 查看生成内容
4. 对比真实仓库目录 → 内容几乎全为幻觉

## 已验证的三个测试

| 测试日期 | 模型 | 结果 | tool_count | API 调用次数 | 提到真实模块名的文档数 |
|---|---|---|---|---|---|
| 2026-05-07 14:46 | LongCat-Flash-Chat | 任务失败(YAML 格式不合法) | 0 | — | — |
| 2026-05-07 15:02 | GLM-4-Flash | 任务"成功",内容幻觉 | 0 | 11 | 0/3 |
| 2026-05-07 15:39 | Claude Sonnet 4.6 | 任务"成功",内容幻觉 | 0 | 6 | 0/5 |

即使换上目前 tool-calling 最强的 Claude Sonnet 4.6,现象依然一致,可证明**与模型能力无关**。

## 期望行为

- `tool_count > 0`,工具 schema 正确随请求发出
- Agent 实际调用 `list_dir`、`read_file`、`search_files` 等工具探索仓库
- 生成的文档**基于真实代码内容**,提到实际的模块、类、函数名
- API Key 请求计数明显高于现在(和 Agent 迭代次数成正比)

## 实际行为

- `tool_count = 0`,请求里 tools 字段空
- 模型在无工具的情况下凭常识和标题猜测生成"看起来合理"的文档
- 文档完全脱离真实代码
- API 调用次数低到不可能完成真实分析

见 `001-工具未绑定到模型-缺陷分析.md`。
